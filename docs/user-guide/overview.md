# Überblick

Das Nutzungshandbuch richtet sich an Endanwender. Ziel ist, dass neue Nutzer die Software schnell und produktiv einsetzen können, ohne zusätzliche Rückfragen stellen zu müssen. Deshalb werden Funktionen nicht nur aufgezählt, sondern in ihren praktischen Nutzen eingeordnet.

## Einstieg

Nach dem Start des Backends leitet die Root-URL auf das Dashboard weiter. Von dort aus sind die wichtigsten Funktionsbereiche direkt erreichbar. Die Oberfläche ist so gedacht, dass Benutzer nicht über technische Begrifflichkeiten nachdenken müssen, sondern über ihre eigentlichen Ziele: Dokumente hochladen, wiederfinden, ordnen, prüfen und bei Bedarf wiederherstellen.

## Hauptfunktionen

- Registrierung, Login und benutzerbezogene Nutzung
- Sichere Ablage hochgeladener Dateien
- OCR-gestützte Volltextsuche
- Kategorien und Schlüsselwörter
- Versionierung
- Favoriten
- Papierkorb mit Wiederherstellung

## Benutzerprinzip

Der Dokumentenmanager ist nicht als überladene Enterprise-Oberfläche gedacht, sondern als klarer Arbeitsraum. Die Software soll typische Vorgänge mit möglichst wenigen Klicks unterstützen. Gleichzeitig ist die Oberfläche eng an die Daten- und Sicherheitslogik gekoppelt. Das bedeutet: Was im Hintergrund als Version, Kategorie oder Papierkorbzustand modelliert ist, erscheint auch in der UI als nachvollziehbare Handlungsmöglichkeit.

## Von der ersten Nutzung zur Routine

Ein neuer Benutzer muss im Regelfall nur drei Dinge verstehen:

1. **Wie lade ich Dokumente hoch?**
2. **Wie finde ich sie später wieder?**
3. **Wie verhindere ich versehentliche Verluste oder Dubletten?**

Genau entlang dieser Fragen ist das Nutzungshandbuch strukturiert.
