# Format & Deliverables

## Bewertungscheckliste

- Landingpage: Produktname/Logo, Value Proposition (3–4 Sätze), Status (Badges)
- Installation & Quick Start: reproduzierbar, ohne Rückfragen, inklusive .env-Erklärung
- Technische Dokumentation: Architekturdiagramm, Datenbankmodell, API-Referenz/Dateistruktur
- Nutzungshandbuch: Use-Cases, Screenshots, Beispiel-Daten
- Technische Strategie: Stack + Alternativen/Trade-offs, Skalierbarkeit, Wartbarkeit, Performance, Workarounds
- Form: korrekte Verlinkungen, saubere Einbindung von Bildern/Diagrammen, konsistentes Design

## GitHub Pages Deployment (GitHub Actions)

1. Repository -> Settings -> Pages
2. Source: GitHub Actions
3. Commit/Push von docs/, mkdocs.yml und Workflow
4. Action baut und deployed automatisch
